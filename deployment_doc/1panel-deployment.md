## 部署前准备

### 1. 服务器要求

- 已安装 1Panel 管理平台
- 服务器可用内存至少 1GB
- 可用磁盘空间至少 2GB

### 2. 本地开发环境

- Node.js 18+
- npm 或 yarn
- Git

### 3. 准备密钥

在部署前，需要生成以下密钥：

```bash
# 生成数据库加密密钥（64位十六进制）
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# 生成 CSRF 密钥（64位十六进制）
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# 生成会话密钥
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

请保存这些密钥，配置环境变量时需要使用。

## 在 1Panel 中创建数据库

### 步骤 1: 进入数据库管理

登录 1Panel，点击左侧菜单 **数据库** → **创建数据库**。

### 步骤 2: 配置数据库

| 配置项 | 推荐值 | 说明 |
|--------|--------|------|
| 数据库类型 | PostgreSQL | 选择 PostgreSQL |
| 数据库名 | `student_leave` | 数据库名称 |
| 用户名 | `student_leave` | 数据库用户名 |
| 密码 | [设置强密码] | 请使用强密码 |
| 端口 | 5432 | 默认端口 |
| 服务限制 | 根据服务器配置 | 可选 |

### 步骤 3: 保存连接信息

创建成功后，记下以下信息：
- 数据库主机：通常是 `127.0.0.1` 或容器名
- 数据库端口：`5432`
- 数据库名：`student_leave`
- 用户名：你设置的用户名
- 密码：你设置的密码

## 编译项目

在本地开发环境中执行以下命令：

```bash
# 1. 进入项目目录
cd /path/to/student_leave

# 2. 安装依赖
npm install

# 3. 构建生产版本
npm run build:release
```

构建完成后，项目根目录下会生成 `dist/` 目录，包含所有部署所需的文件。

## 上传部署文件

### 方式 1: 使用 SCP 上传（推荐）

```bash
# 在本地执行，将 dist 目录上传到服务器
scp -r dist/ user@your-server-ip:/opt/student_leave

# 示例
scp -r dist/ root@192.168.1.100:/opt/student_leave
```

### 方式 2: 使用 1Panel 文件管理

1. 登录 1Panel
2. 点击左侧菜单 **主机** → **文件**
3. 进入 `/opt/` 目录
4. 创建新目录 `student_leave`
5. 进入 `student_leave` 目录
6. 点击上传按钮，上传 `dist/` 目录下的所有文件

### 方式 3: 使用 SFTP 客户端

使用 FileZilla、WinSCP 等 SFTP 客户端连接服务器，上传文件到 `/opt/student_leave`。

## 在 1Panel 中创建运行环境

### 步骤 1: 进入容器管理

登录 1Panel，点击左侧菜单 **容器** → **运行环境** → **创建运行环境**。

### 步骤 2: 基本配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **名称** | `student-leave-app` | 运行环境名称 |
| **应用** | Node.js | 选择应用类型 |
| **版本** | 18 或 20 | Node.js 版本 |
| **项目目录** | `/opt/student_leave` | 指向上传的文件目录 |
| **启动命令** | `node server.js` | 使用 standalone 输出的启动方式 |
| **容器名称** | `student-leave-app` | 容器标识 |
| **包管理器** | 无需选择 | 已编译的项目不需要 |

### ⚠️ 重要提示

1. **不要配置 "安装命令"** - 部署包已包含所有依赖（node_modules）
2. **不要配置 "构建命令"** - 项目已经编译好
3. **启动命令** 直接使用 `node server.js`

如果配置了 npm install，会出现 "npm warn deprecated" 警告，但不影响运行。建议不要配置安装命令，避免不必要的警告。

### 步骤 3: 端口映射

点击 **端口映射** 添加映射：

| 容器端口 | 主机端口 | 协议 |
|----------|----------|------|
| 3000 | 3000 或 8080 | TCP |

> **说明**：3000 是 Next.js 默认端口，可以映射到主机的 3000 或 8080 端口。

### 步骤 4: 卷挂载（可选但推荐）

点击 **卷挂载** 添加挂载：

| 容器目录 | 主机目录 | 说明 |
|----------|----------|------|
| `/app/backups` | `/opt/student_leave/backups` | 备份文件持久化 |
| `/app/logs` | `/opt/student_leave/logs` | 日志文件持久化 |

### 步骤 5: 创建运行环境

点击 **确认** 创建运行环境，等待容器启动。

## 配置环境变量

### 步骤 1: 进入环境变量配置

在运行环境列表中，找到 `student-leave-app`，点击 **环境变量**。

### 步骤 2: 添加环境变量

点击 **添加环境变量**，逐个添加以下变量：

```env
# Node 环境
NODE_ENV=production

# 数据库连接（根据你的实际配置修改）
# 格式: postgresql://用户名:密码@主机:端口/数据库名
POSTGRES_URL=postgresql://student_leave:你的数据库密码@127.0.0.1:5432/student_leave

# 数据库加密密钥（必填，64位十六进制）
# 使用前面生成的密钥
DB_ENCRYPTION_KEY=你的64位十六进制密钥

# CSRF 密钥（必填，64位十六进制）
# 使用前面生成的密钥
CSRF_SECRET=你的64位十六进制密钥

# 会话密钥（必填）
# 使用前面生成的密钥
SESSION_SECRET=你的会话密钥

# 应用 URL
# 修改为你的服务器 IP 和端口
NEXT_PUBLIC_APP_URL=http://你的服务器IP:端口
```

### 步骤 3: 重启容器

环境变量配置完成后，点击 **重启** 使配置生效。

## 初始化数据库

### 方式 1: 通过容器终端（推荐）

1. 在 1Panel 中，进入 **容器** → **容器**
2. 找到 `student-leave-app` 容器，点击 **终端**
3. 在终端中执行以下命令：

```bash
# 进入应用目录
cd /app

# 执行数据库迁移
npm run db:migrate
```

等待迁移完成，看到类似 `Migration completed successfully` 的提示。

### 方式 2: 通过计划任务

1. 在 1Panel 中，进入 **计划任务** → **创建计划任务**
2. 任务类型选择：**Shell 脚本**
3. 任务名称：`初始化学生请假系统数据库`
4. 执行周期：**仅执行一次**
5. 脚本内容：

```bash
docker exec student-leave-app sh -c "cd /app && npm run db:migrate"
```

6. 点击 **执行** 立即运行任务。

## 后期更新

### 更新应用代码（不涉及数据库结构变化）

当项目代码更新，但数据库结构没有变化时：

```bash
# 1. 本地重新编译
cd /path/to/student_leave
npm run build:release

# 2. 上传到服务器覆盖旧文件
scp -r dist/* user@your-server:/opt/student_leave/

# 3. 在 1Panel 中重启容器
# 进入 容器 → student-leave-app → 重启
```

### 更新应用代码（涉及数据库结构变化）

当项目更新包含数据库结构变化时：

```bash
# 1. 本地重新编译
npm run build:release

# 2. 上传到服务器
scp -r dist/* user@your-server:/opt/student_leave/

# 3. 在 1Panel 中进入容器终端
# 执行数据库迁移
npm run db:migrate

# 4. 重启容器
```

### 备份数据

建议定期备份数据库：

1. 在 1Panel 中，进入 **数据库**
2. 找到 `student_leave` 数据库
3. 点击 **备份** → **创建备份**
4. 选择备份位置和保留时间

## 常见问题

### 1. 容器启动失败

**症状**：容器状态显示 "已停止" 或 "重启中"

**解决方案**：

1. 检查启动命令是否为 `node server.js`
2. 检查项目目录是否正确（`/opt/student_leave`）
3. 查看容器日志排查错误

### 2. 数据库连接失败

**症状**：应用提示 "数据库连接失败"

**解决方案**：

1. 检查 `POSTGRES_URL` 环境变量是否正确
2. 确认数据库容器正在运行
3. 确认网络连接（测试数据库端口）
4. 检查数据库用户名和密码

### 3. 端口无法访问

**症状**：浏览器无法访问应用

**解决方案**：

1. 检查 1Panel 防火墙规则，确保端口开放
2. 检查服务器安全组（云服务器）
3. 确认端口映射配置正确
4. 使用 `netstat -tlnp` 检查端口监听状态

### 4. 静态资源加载失败

**症状**：页面样式错乱或图片无法显示

**解决方案**：

1. 确认 `public/` 目录已正确上传到服务器
2. 检查 `NEXT_PUBLIC_APP_URL` 环境变量
3. 清除浏览器缓存重试

### 5. 数据库迁移失败

**症状**：执行 `npm run db:migrate` 报错

**解决方案**：

1. 确认数据库连接正常
2. 检查数据库用户是否有权限创建表
3. 查看迁移日志定位具体问题
4. 如果表已存在，可能需要手动清理

### 6. 容器内存不足

**症状**：容器频繁重启或应用响应缓慢

**解决方案**：

1. 在 1Panel 中调整容器内存限制
2. 检查服务器内存使用情况
3. 清理不必要的容器和镜像
